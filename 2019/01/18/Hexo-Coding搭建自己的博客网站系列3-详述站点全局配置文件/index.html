Hexo 目录如下图

这是Blog文件的目录。从上到下依次解释

  1. node_modules:hexo整个项目的环境依赖，hexo初始化时自动生成的。
  2. public:存放的是整个项目的页面，样式，及逻辑代码。
  3. scaffolds:scaffolds是脚手架的意思，当你新建一篇文章(hexo new page ‘title’)的时候，hexo是根据这个目录下的文件进行构建的。
  4. source:这个目录很重要，新建的文章都是在保存在这个目录下的_posts 。_posts 目录下是一个个 markdown 文件。
  5. themes:是网站的主题目录，Hexo有很好的主题扩展能力，开源主题也很丰富，该目录下每一个子目录就是一个主题。
  6. _config.yml:站点全局配置文件。
